
### `Definition`

The **testing (QA) environment** is used to verify that the application behaves correctly and meets requirements before release. It is more controlled and stable than development.

---
### `Key Ideas`

- **Validation Focus:**  
    Used for unit tests, integration tests, and system tests.
    
- **Closer to Production:**  
    Environment configuration starts to resemble production.
    
- **Controlled Changes:**  
    Code is deployed intentionally for testing purposes.
    
- **Automated Testing:**  
    Often integrated into CI pipelines.
    
- **Bug Detection:**  
    Primary place to find defects before release.
